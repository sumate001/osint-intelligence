# OSINT//DESK — System Architecture
## เอกสารสถาปัตยกรรมระบบฉบับสมบูรณ์

## 1. ภาพรวมสถาปัตยกรรม

OSINT//DESK ออกแบบตามหลัก separation of concerns โดยแบ่งระบบเป็นสามชั้นที่ทำงานอิสระจากกัน ได้แก่ ชั้นอัตโนมัติที่ทำงานเบื้องหลัง ชั้นพื้นที่ทำงานของผู้ใช้ และชั้นบริหารจัดการ ทั้งสามชั้นใช้ทรัพยากรร่วมกันผ่าน service layer กลาง

ระบบทั้งหมดทำงานบนคอนเทนเนอร์ผ่าน Docker Compose ทำให้ติดตั้งได้ด้วยคำสั่งเดียวและย้ายระหว่างเซิร์ฟเวอร์ได้ง่าย ไม่มี dependency ที่ผูกกับผู้ให้บริการคลาวด์รายใดรายหนึ่ง

หลักการออกแบบสำคัญคือข้อมูลทั้งหมดอยู่ในเซิร์ฟเวอร์ของห้องข่าวเอง ปัญญาประดิษฐ์รันในเครื่อง และการค้นหาผ่าน proxy ที่ไม่ถูกติดตาม ทำให้งานข่าวอ่อนไหวปลอดภัยจากการรั่วไหล

## 2. องค์ประกอบของระบบ

### 2.1 ชั้นนำเสนอ (Presentation Layer)

ส่วนติดต่อผู้ใช้สร้างด้วย Next.js และ React จัดรูปแบบด้วย Tailwind CSS แสดงผลกราฟความสัมพันธ์ด้วย vis.js และแผนภูมิข้อมูลด้วย Chart.js ออกแบบให้ทำงานได้ดีทั้งบนเดสก์ท็อปและแท็บเล็ต รองรับการใช้งานในห้องข่าวที่ต้องดูหลายหน้าจอพร้อมกัน

### 2.2 ชั้นบริการ (API Layer)

แกนกลางของระบบคือ FastAPI ที่เขียนด้วย Python จัดการการยืนยันตัวตนด้วย JWT และควบคุมสิทธิ์การเข้าถึงตามบทบาท แบ่งผู้ใช้เป็นบรรณาธิการ นักวิเคราะห์ และนักข่าว แต่ละบทบาทเห็นและทำได้ต่างกัน

งานที่ใช้เวลานานเช่นการสแกนข้อมูลถูกจัดการผ่านคิวงาน Celery ร่วมกับ Redis ทำให้ส่วนติดต่อผู้ใช้ไม่ค้างระหว่างรอผลและสามารถตั้งเวลาทำงานอัตโนมัติได้

### 2.3 ชั้นค้นหาและวิจัย (Search & Research Layer)

นี่คือหัวใจที่แยก OSINT//DESK ออกจากระบบจัดการเนื้อหาทั่วไป ประกอบด้วยสองส่วนที่ทำงานคนละระดับ

SearXNG เป็นเสิร์ชเอนจินรวมที่ส่งคำค้นไปยังเสิร์ชเอนจินหลายตัวพร้อมกัน รวมผลกลับมาโดยไม่ถูกติดตาม ใช้ในงานอัตโนมัติที่ต้องค้นหาจำนวนมาก เช่น การเฝ้าระวังรายการติดตามและการป้อนข้อมูลให้ระบบคัดกรอง

Perplexica เป็นผู้ช่วยวิจัยด้วยปัญญาประดิษฐ์ที่ใช้ SearXNG เป็นฐาน แต่เพิ่มความสามารถในการสังเคราะห์คำตอบพร้อมแหล่งอ้างอิง ใช้เมื่อนักวิเคราะห์ต้องการค้นหาข้อมูลเชิงลึกแบบถาม-ตอบระหว่างการสืบสวน

### 2.4 ชั้นเครื่องมือ OSINT (OSINT Engine Layer)

SpiderFoot ทำหน้าที่เป็นเครื่องมือสแกนข้อมูลอัตโนมัติ ส่งคำค้นไปยังแหล่งข้อมูลหลายสิบแหล่งพร้อมกัน ทั้งข้อมูลโดเมน ระบบชื่อโดเมน และฐานข้อมูลสาธารณะ แล้วประมวลผลความสัมพันธ์เก็บไว้

ระบบตรวจสอบสื่อใช้ ExifTool สำหรับดึงข้อมูลกำกับไฟล์ ใช้ Wayback Machine สำหรับตรวจสอบการเผยแพร่ครั้งแรก และใช้ faster-whisper สำหรับถอดเสียงวิดีโอเพื่อตรวจสอบเนื้อหา

### 2.5 ชั้นปัญญาประดิษฐ์ (AI Layer)

โมเดลภาษาทั้งหมดรันในเครื่องผ่าน Ollama โดยใช้ qwen3:8b เป็นโมเดลหลักสำหรับการสกัดข้อมูลสำคัญ การให้คะแนนข่าว และการร่างรายงาน การรันในเครื่องทำให้ข้อมูลอ่อนไหวไม่ต้องส่งออกไปยังบริการภายนอกและไม่มีค่าใช้จ่ายต่อการเรียกใช้

### 2.6 ชั้นข้อมูล (Data Layer)

ระบบใช้ฐานข้อมูลสามชนิดตามลักษณะข้อมูล PostgreSQL เก็บข้อมูลหลักเช่นคดี หลักฐาน และผู้ใช้ Neo4j เก็บความสัมพันธ์ระหว่างเอนทิตีในรูปแบบกราฟ ซึ่งสืบค้นความเชื่อมโยงซับซ้อนได้เร็วกว่าฐานข้อมูลแบบตารางมาก และ Meilisearch ทำดัชนีค้นหาข้อความเต็มข้ามทุกคดี

ไฟล์เช่นภาพหน้าจอ หลักฐาน และวิดีโอ เก็บใน MinIO ซึ่งเข้ากันได้กับมาตรฐาน S3 ทำให้ย้ายไปคลาวด์ได้ในอนาคตหากต้องการ

### 2.7 ชั้นอัตโนมัติ (Automation Layer)

n8n จัดการกระบวนการอัตโนมัติทั้งหมด ทั้งการดูดข้อมูลจาก RSS การรับข้อมูลจาก GDELT และ ACLED และการตั้งเวลาสแกน เชื่อมต่อส่วนต่างๆ ของระบบเข้าด้วยกันโดยไม่ต้องเขียนโค้ดมาก

### 2.8 ชั้นเฝ้าระวังระบบ (Monitoring Layer)

Grafana ร่วมกับ Prometheus เฝ้าดูสุขภาพของทุกบริการ แสดงสถานะงานสแกน การใช้ทรัพยากร และแจ้งเตือนเมื่อมีปัญหา ทำให้ผู้ดูแลระบบรู้ทันทีหากบริการใดมีปัญหา

## 3. การไหลของข้อมูล

### 3.1 กระบวนการอัตโนมัติ

เมื่อข่าวใหม่เข้ามา n8n ดึงจาก RSS ทุกห้านาที แยกหัวข้อ แหล่งที่มา และเวลา แล้วส่งเข้าคิวใน Redis จากนั้น Celery worker รับงานส่งต่อให้โมเดลภาษาให้คะแนนตามเกณฑ์เจ็ดด้าน สกัดเอนทิตีที่เกี่ยวข้อง และจัดระดับความสำคัญ ผลลัพธ์เก็บใน PostgreSQL และหากคะแนนถึงเกณฑ์จะแจ้งเตือนทันที

การเฝ้าระวังสภาพแวดล้อมทำงานคู่ขนาน SearXNG ค้นหาคำสำคัญของแต่ละประเด็นทุกชั่วโมง โมเดลภาษาจัดประเภทว่าตรงกับประเด็นใด คำนวณความเข้มของสัญญาณเทียบกับค่าเฉลี่ยสามสิบวัน และแจ้งเตือนหากพบความผิดปกติ

การเฝ้าระวังรายการติดตามใช้ SpiderFoot สแกนทุกเอนทิตีตามรอบเวลา เปรียบเทียบกับผลครั้งก่อน หากพบข้อมูลใหม่จะแจ้งเตือนผู้รับผิดชอบ

### 3.2 กระบวนการที่ผู้ใช้ควบคุม

นักข่าวเปิดระบบมาเห็นผลการคัดกรองที่ระบบทำไว้แล้ว เลือกข่าวที่ต้องการสืบสวน ระบบเปิดพื้นที่ทำงานพร้อมเริ่มสแกน SpiderFoot นักวิเคราะห์ดูกราฟความสัมพันธ์ ค้นหาข้อมูลเพิ่มผ่าน Perplexica เพิ่มหลักฐานพร้อมแหล่งอ้างอิง และเมื่อพร้อมก็ส่งต่อไปเขียนรายงาน

## 4. ความปลอดภัยและความเป็นส่วนตัว

ระบบออกแบบโดยคำนึงถึงความอ่อนไหวของงานข่าวเป็นหลัก

ข้อมูลทั้งหมดอยู่ในเซิร์ฟเวอร์ของห้องข่าว ไม่มีการส่งออกไปยังบริการภายนอก การค้นหาผ่าน SearXNG ถูกปกปิดที่อยู่ปลายทาง การยืนยันตัวตนใช้ JWT และการเข้าถึงทุกส่วนควบคุมตามบทบาท

สำหรับงานที่อ่อนไหวเป็นพิเศษ ระบบรองรับการทำงานผ่านเครือข่ายนิรนามและเครื่องเสมือนแยก เพื่อป้องกันการเปิดเผยตัวตนของผู้สืบสวน

ทุกการกระทำในระบบถูกบันทึกไว้ ทำให้ตรวจสอบย้อนหลังได้ว่าใครเข้าถึงข้อมูลใดเมื่อไหร่ ซึ่งสำคัญทั้งด้านความปลอดภัยและด้านความรับผิดชอบทางบรรณาธิการ

---

## 5. ความต้องการด้านฮาร์ดแวร์

สำหรับการเริ่มต้นในสามเดือนแรก แนะนำเซิร์ฟเวอร์ที่มีหน่วยประมวลผลแปดแกน หน่วยความจำสามสิบสองกิกะไบต์ และพื้นที่จัดเก็บแบบ SSD ห้าร้อยกิกะไบต์ รันระบบปฏิบัติการ Ubuntu

สำหรับการใช้งานเต็มรูปแบบ แนะนำหน่วยประมวลผลสิบหกถึงสามสิบสองแกน หน่วยความจำหกสิบสี่กิกะไบต์ พื้นที่จัดเก็บสองเทระไบต์ และหากต้องการประมวลผลปัญญาประดิษฐ์หนักควรมีหน่วยประมวลผลกราฟิกเสริม

โมเดลภาษาใช้หน่วยความจำมากที่สุด การมีหน่วยประมวลผลกราฟิกช่วยเร่งความเร็วได้มาก แต่ระบบทำงานบนหน่วยประมวลผลกลางล้วนได้เช่นกัน

## 6. รายการบริการทั้งหมด

ระบบประกอบด้วยบริการที่รันในคอนเทนเนอร์ดังนี้ ตัวแทนพร็อกซีย้อนกลับ nginx ส่วนติดต่อผู้ใช้ Next.js แกนบริการ FastAPI ตัวประมวลผลงาน Celery ฐานคิว Redis ฐานข้อมูลหลัก PostgreSQL ฐานข้อมูลกราฟ Neo4j ดัชนีค้นหา Meilisearch ที่เก็บไฟล์ MinIO เครื่องมือสแกน SpiderFoot เสิร์ชเอนจินรวม SearXNG ผู้ช่วยวิจัย Perplexica ระบบอัตโนมัติ n8n เซิร์ฟเวอร์ปัญญาประดิษฐ์ Ollama และระบบเฝ้าระวัง Grafana กับ Prometheus

ทั้งหมดนิยามไว้ในไฟล์ docker-compose เดียว ติดตั้งและอัปเดตได้ด้วยคำสั่งเดียว
